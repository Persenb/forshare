Class 1 — C++ 基础与递归入门

1. score_sort — 结构体多关键字排序

问题：按语文成绩升序排，相同时按数学成绩升序排。

思想：自定义比较函数

struct score {
    int chinese;
    int math;
};

bool comp(score x, score y) {
    if (x.chinese != y.chinese)
        return x.chinese < y.chinese;    // 主关键字
    else
        return x.math < y.math;           // 次关键字
}

sort(a, a + n, comp);

要点：comp 必须返回严格弱序（strict weak ordering），即 return a < b 而不是 a <= b。

2. abs_value_sort — 按绝对值排序

思想：自定义排序 + abs()

bool comp(int x, int y) {
    return abs(x) > abs(y);    // 绝对值降序
}

sort() 默认升序（less），降序用 greater<int>() 或自定义 comp。

3. count — 约瑟夫问题（模拟）

问题：n 个人围一圈，数到 k 的人出列，求最后剩下的人。

思想：循环数组模拟

用 bool 数组标记是否出列
count 指针循环走，跳过已出列的人
数到 k → 标记出列，人数 -1
直到只剩 1 人
while (n2 != 1) {
    if (!person[count]) {        // 还活着
        if (k2 == k) {           // 数到了
            person[count] = 1;   // 出列
            n2--;
            k2 = 0;
        }
        k2++;
    }
    count++;
    if (count > n) count = 1;    // 环形
}

复杂度：O(nk) — 每次淘汰一个人要数 k 步。

进阶：约瑟夫问题可以用递推公式 O(n) 解决：f(1)=0; f(i)=(f(i-1)+k)%i。

4. ants — 蚂蚁掉杆问题

问题：杆上有若干蚂蚁，每只蚂蚁在两端之间爬，相遇则掉头，求所有蚂蚁都掉下去的最短和最长时间。

思想：相遇等价于穿过（关键思维）

两只蚂蚁相遇掉头 ↔ 相当于互相穿过（个体不重要，重要的是位置）
                       ↓
每只蚂蚁的掉落时间就是 min(p, len-p) 或 max(p, len-p)
最短时间：所有蚂蚁取 min 值中的最大值（因为要等最慢的那只）
最长时间：所有蚂蚁取 max 值中的最大值
for (int i = 1; i <= t; i++) {
    int p; cin >> p;
    a[i].ls = min(p, len - p);   // 这只蚂蚁的最短掉落时间
    a[i].lm = max(p, len - p);   // 这只蚂蚁的最长掉落时间
}
int minT = 0, maxT = 0;
for (int i = 1; i <= t; i++) {
    if (a[i].ls > minT) minT = a[i].ls;   // 短板效应：最慢的那个决定了总时间
    if (a[i].lm > maxT) maxT = a[i].lm;
}

核心洞察：蚂蚁相遇掉头 → 等价穿透。每只蚂蚁独立计算到两端的时间即可。

5. fibonaci_recurve — 斐波那契（朴素递归）

long long fib(int t) {
    if (t == 0 || t == 1) return 1;
    return fib(t - 1) + fib(t - 2);
}

问题：大量重复计算。fib(5) 调用树：

                    fib(5)
                  /        \
             fib(4)         fib(3)
            /      \        /    \
        fib(3)    fib(2) fib(2)  fib(1)
       /    \
    fib(2) fib(1)

复杂度：O(2ⁿ) — 指数爆炸，n=50 就跑不动了。

6. fibonaci — 斐波那契（递推 + 高精度大数）

思想：数组模拟大数 + 递推

用数组 a[], b[], c[] 模拟大整数，每个元素存一位数字
a=1, b=1 → 循环做 a+b → 更新 a=b, b=c
void do_plus() {
    int temp = 0;
    for (int i = 1; i <= size; i++) {
        c[i] = a[i] + b[i] + temp;
        temp = c[i] / 10;
        c[i] %= 10;
    }
    if (temp) c[++size] = temp;   // 进位，位数 +1
}

对比：

版本 方法 可算到 原因
递归版 fib(n-1)+fib(n-2) n ≤ 40 指数爆炸
递推版 循环迭代 n ≤ 90 long long 溢出
大数版 数组模拟高精度 n 任意大 手算进位

7. hanno_tower — 汉诺塔（递归 + 步骤输出）

问题：把 n 个盘子从 A 移到 C，每次移一个，大盘不能压小盘。

思想：递归分治

要把 n 个盘子从 src 移到 dest（借助 mid）：
    ① 把上面 n-1 个盘子从 src 移到 mid（借助 dest）
    ② 把最底下的盘子从 src 移到 dest     ← 1 步
    ③ 把 mid 上的 n-1 个盘子移到 dest（借助 src）
void move(int src, int mid, int dest, int t) {
    if (t == 1) {
        cout << "from " << src << " to " << dest << "\n";
        return;
    }
    move(src, dest, mid, t - 1);       // ①
    cout << "from " << src << " to " << dest << "\n";  // ②
    move(mid, src, dest, t - 1);       // ③
}

步数公式：f(n) = 2*f(n-1) + 1 → f(n) = 2ⁿ - 1

8. hannotpwer_simp — 汉诺塔（仅计算步数）

long long move(int step) {
    if (step == 1) return 1;
    return move(step - 1) * 2 + 1;
}

递归计算 2ⁿ - 1，n 较大时会溢出，此时需用大数或公式直接算 (1LL << n) - 1。

9. median — 两两差值的中位数（暴力版）

问题：求数组所有数对差值的中位数。

思想：暴力枚举所有差值 → 排序 → 取中位数

for (int i = 1; i <= N; i++)
    for (int j = i + 1; j <= N; j++)
        a[t++] = abs(b[i] - b[j]);

sort(a, a + t);

复杂度：O(n² log n) — class2 中的 median_optimized 用”二分答案+双指针”优化到了 O(n log n)。

10. balanced_lineup — 区间最值差（暴力版）

问题：多次询问区间 [l, r] 的最大值与最小值之差。

思想：每次查询遍历区间

for (int j = b[i].l; j <= b[i].r; j++) {
    if (a[j] < min) min = a[j];
    if (a[j] > max) max = a[j];
}

复杂度：O(nq) — class3 中的 balanced_lineupSTtable 用 ST 表优化到了 O(n log n + q)。

对比学习：同一个问题，class1 用暴力（理解问题），class3 用 ST 表（优化性能）。这是很好的学习路径——先暴力，再优化。

Class 2 — 双指针与二分查找

1. boring_count — 滑动窗口统计合法子串

问题：给定字符串 s 和整数 k，统计每个字符出现次数不超过 k 的子串个数。

思想：滑动窗口（双指针）

右指针不断向右扩展 → 窗口变大 → 条件被破坏
                                 ↓
                    左指针右移收缩 → 条件恢复
                                 ↓
                        统计以右指针结尾的合法子串数

关键代码：

for (int right = 0; right < len; right++) {
    num[s[right] - 'a']++;                          // 右指针扩展
    while (num[s[right] - 'a'] > k) {               // 不合法 → 收缩
        num[s[left] - 'a']--;
        left++;
    }
    ans += (right - left + 1);                      // 统计以 right 结尾的子串
}

核心技巧：ans += (right - left + 1) — 每次加入一个新右端点，新增的合法子串数就是当前窗口的长度（即以 right 结尾、左端点可取 left ~ right 的任一位置）。

2. can_you_find_it — 三数列求和判定

问题：三个数列 A、B、C，问能否从 A 和 B 中各选一个数、从 C 中选一个数，使三数之和等于目标值。

思想：预处理 + 二分查找

A × B 的所有两两和 → 存入数组并排序
                     ↓
        对每个 c ∈ C，二分查找 target - c 是否在两两和数组中

复杂度：O(n² log n)，暴力是 O(n³)。

关键代码：

// 预处理 A 和 B 的所有两两和
for (int i = 1; i <= m; i++) {
    int t; cin >> t;
    for (int j = 1; j <= l; j++)
        b[(i-1)*l + j] = t + a[j];     // 依次排列
}
sort(b + 1, b + l*m + 1);               // 排序

// 查询：对每个 c 做二分查找
for (int i = 1; i <= n; i++) {
    if (binary_search(b + 1, b + l*m + 1, t - c[i])) {
        f = true; break;
    }
}

适用场景：当暴力需要多重循环时，考虑能否预处理部分组合，配合二分加速。

3. median_optimized — 两两差值的中位数

问题：给定数组，求所有数对差值的中位数。

思想：二分答案 + 双指针验证

排序数组
    ↓
二分猜测答案 mid
    ↓
双指针统计差值 ≤ mid 的数对个数 cnt
    ↓
cnt ≥ 中位数位置？ → 缩小右边界 / 放大左边界

关键代码：

sort(a + 1, a + 1 + n);

int left = 0, right = a[n] - a[1];
while (left <= right) {
    int mid = left + (right - left) / 2;   // 防止溢出
    if (check(mid, k)) {                   // check 用双指针
        ans = mid;
        right = mid - 1;
    } else {
        left = mid + 1;
    }
}

双指针 check 的精髓：每个数做一次被减数，j 指针不回退，总体 O(n)。

bool check(int mid, int k) {
    int cnt = 0, j = 2;
    for (int i = 1; i <= n; i++) {
        while (j <= n && a[j] - a[i] <= mid) j++;
        cnt += j - i - 1;
    }
    return cnt >= k;
}

为什么 j 可以不回退？因为 i 增大时 a[i] 变大，a[j] - a[i] 变小，之前的 j 位置已经满足条件了。

Class 3 — 栈、单调栈与 ST 表

1. River_hopscotch — 移除石头最大化最小跳跃距离

问题：河中有 n 块石头，移除 m 块后，使剩余石头之间的最小跳跃距离最大化。

思想：二分答案 + 贪心模拟

二分答案 mid（猜测最小跳跃距离）
    ↓
从头遍历石头，遇到距离 < mid 的就"移除"
（用 a[i] = a[i-1] 模拟移除）
    ↓
检查移除总数是否 ≤ m

关键点： - 每次 check 完要用 memcpy 还原数组 - 最后一块石头的距离需要单独检查 - 二分边界：left = 最小间距，right = 最大间距

代码模式：

bool check(int mid) {
    int removed = 0;
    for (int i = 0; i <= n; i++) {
        if (a[i+1] - a[i] < mid) {
            // 移除第 i+1 块石头
            removed++;
        }
    }
    return removed <= m;
}

while (left <= right) {
    mid = (left + right) / 2;
    if (check(mid)) { ans = mid; left = mid + 1; }  // 尝试更大的答案
    else { right = mid - 1; }
}

2. balanced_lineupSTtable — ST表（Sparse Table）

问题：多次查询区间最大值与最小值之差。

思想：倍增预处理，O(1) 查询

定义 st[k][i] = 从 i 开始长度 2^k 的区间的最值
                        ↓
st[0][i] = a[i]        （长度为 1）
st[k][i] = max(st[k-1][i], st[k-1][i + 2^(k-1)])
                       （两个半段合并）

查询：区间 [l, r]，长度 len

int k = log2(len);   // 不大于 len 的最大 2 的幂
return max(st[k][l], st[k][r - (1<<k) + 1]);

优点：查询极快 O(1)
缺点：不支持修改（动态更新需用线段树）

log 表预处理：

logtable[1] = 0;
for (int i = 2; i <= n; i++)
    logtable[i] = logtable[i/2] + 1;

3. expression_evaluation — 表达式求值（加减乘）

问题：计算只含 +、* 的表达式，结果模 10000。

思想：延迟计算 / 累乘器

遍历表达式：
  · 遇到数字 → 拼接到 num
  · 遇到运算符或结尾 →
        cur = (cur * num) % 10000    ← 乘到累乘器
        num = 0
        如果是加号或结尾 →
            ans = (ans + cur) % 10000
            cur = 1                  ← 重置累乘器
遍历 1+2*3+4 cur ans
遇到 +（num=1） 1×1=1 0+1=1
遇到 *（num=2） 1×2=2 1
遇到 +（num=3） 2×3=6 1+6=7
结尾（num=4） 1×4=4 7+4=11

技巧：初始化 cur=1，这样加法操作时 cur * num = num，不影响结果。

4. largest_rectangle — 单调栈求最大矩形

问题：直方图中找面积最大的矩形。

思想：单调递增栈

每个柱子的高度能向左右延伸多远？
    · 左边第一个比它矮的柱子 + 1
    · 右边第一个比它矮的柱子 - 1
                    ↓
    单调栈维护递增高度，遇到矮柱子就弹出结算

结算时机：当前柱子高度 < 栈顶高度

while (!st.empty() && cur < heights[st.top()]) {
    int h = heights[st.top()];
    st.pop();
    int left = st.empty() ? 0 : st.top() + 1;   // 左边界
    int right = i - 1;                           // 右边界
    maxArea = max(maxArea, (long long)h * (right - left + 1));
}

为什么要加虚拟柱子（高度0）？让所有柱子最终都被弹出结算。

时间复杂度：O(n) — 每个元素入栈一次，出栈最多一次。

附：二分查找常见用法模板

二分搜索是所有算法中最基础也最易错的技巧。关键在于明确搜索区间和下一步缩小区间的方向。下面是 C++ 竞赛中最常用的 6 种二分模板。

模板 1：标准二分查找（精确值）

用途：在有序数组中找 target，存在返回下标，否则返回 -1。

int binarySearch(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;   // [left, right] 闭区间
    while (left <= right) {                  // 终止条件：left > right
        int mid = left + (right - left) / 2; // 防溢出写法
        if (nums[mid] == target) return mid;
        else if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}

记忆口诀：left <= right，mid 用 left + (right-left)/2，相等直接返回。

模板 2：二分查找左边界（第一个 ≥ target）

用途：在有序数组中找第一个 ≥ target 的元素下标（即 std::lower_bound）。

int lowerBound(vector<int>& nums, int target) {
    int left = 0, right = nums.size();       // [left, right) 左闭右开
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] >= target)
            right = mid;                     // 左边界在左边（含 mid）
        else
            left = mid + 1;                  // 左边界在右边
    }
    return left;                             // left 就是第一个 ≥ target 的位置
}

常见变体：

需求 写法
第一个 ≥ target lower_bound / 上面模板
第一个 > target upper_bound / 把 >= 改成 >
最后一个 < target lower_bound 结果 - 1
最后一个 ≤ target upper_bound 结果 - 1

模板 3：二分查找右边界（最后一个 ≤ target）

用途：在有序数组中找最后一个 ≤ target 的元素下标。

int upperBound(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] <= target)
            left = mid + 1;                  // 右边界在右边，压缩左边界
        else
            right = mid - 1;                 // 右边界在左边
    }
    return right;                            // right 就是最后一个 ≤ target 的位置
}

注意：这个模板返回的是闭区间 [left, right] 中的 right，当 right < 0 时表示不存在。

模板 4：二分答案 — 最小值最大化

用途：求”最大值尽可能大”的问题，如跳石头（最小跳跃距离最大化）。

// check(x): 能否让最小值 ≥ x
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (check(mid)) {
        ans = mid;          // mid 可行，记录答案
        left = mid + 1;     // 尝试更大的答案
    } else {
        right = mid - 1;    // mid 不可行，缩小
    }
}

典型问题：跳石头、牛棚距离、分割数组的最大值。

判断方向：left 往大推（找最大的可行值），right 往小缩。

模板 5：二分答案 — 最大值最小化

用途：求”最小值尽可能小”的问题。

// check(x): 能否让最大值 ≤ x
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (check(mid)) {
        ans = mid;          // mid 可行，记录答案
        right = mid - 1;    // 尝试更小的答案
    } else {
        left = mid + 1;     // mid 不可行，放大
    }
}

典型问题：木棍切割、抄写书、机器人走迷宫。

方向记忆：

模式 check 含义 可行后往哪走
最小值最大化 能否让最小值 ≥ mid left = mid + 1（加大）
最大值最小化 能否让最大值 ≤ mid right = mid - 1（减小）

模板 6：浮点数二分

用途：求平方根、解方程等需要精度的场景。

double floatBinarySearch(double target) {
    double left = 0, right = target;
    for (int i = 0; i < 100; i++) {         // 迭代 100 次，精度足够
        double mid = (left + right) / 2;
        if (mid * mid < target)             // 条件判断
            left = mid;
        else
            right = mid;
    }
    return right;                           // left 和 right 已足够接近
}

两种写法：

// 写法 A：固定迭代次数（推荐，无需考虑精度）
for (int i = 0; i < 100; i++) { ... }

// 写法 B：精度控制（小心死循环）
while (right - left > 1e-7) { ... }

为什么推荐固定 100 次？ 浮点数精度控制容易因精度设太高导致死循环，100 次迭代对 double 来说足够了（每次区间减半，2⁻¹⁰⁰ ≈ 10⁻³⁰）。

STL 中的二分全家桶

#include <algorithm>

bool f = binary_search(a, a + n, x);          // 是否存在 x（返回 bool）

int* p = lower_bound(a, a + n, x);            // 第一个 ≥ x 的指针
int* q = upper_bound(a, a + n, x);            // 第一个 > x 的指针

int idx  = lower_bound(a, a + n, x) - a;      // 转换为下标
int cnt  = upper_bound(a, a + n, x) - lower_bound(a, a + n, x);  // x 的出现次数

常见组合：

需求 写法
找 x 是否存在 binary_search
找 x 第一次出现位置 lower_bound
统计 x 出现次数 upper_bound - lower_bound
找第一个 > x 的元素 upper_bound
找最后一个 ≤ x 的元素 upper_bound 的前一个位置

二分易错点自查清单

易错点 正确做法
mid = (left+right)/2 溢出 改为 left + (right-left)/2
死循环（左边界不更新） 确认每次循环 left 或 right 一定变化
区间混淆（开闭区间不一致） 统一用一种区间并保持全部代码一致
浮点数精度过高导致死循环 用固定迭代次数（100次）代替精度判断
check 函数反了 先确认”可行”的含义再写 true/false 的分支
序号从 1 开始/从 0 开始混用 统一偏移量，STL 用 0-base，手写可以保持 1-base

Class 4 — DFS 连通块、二叉树与单调队列

1. lake_counting — DFS 洪水填充（Flood Fill）

问题：二维网格中找水洼（连通块）个数，8 方向连通。

思想：DFS 标记法

遍历网格 → 遇到 'W' → DFS 将它所在的整个连通块都标记为 '.'
                      → 连通块计数 +1

方向遍历技巧：两层循环覆盖 8 个方向

for (int dx = -1; dx <= 1; dx++)
    for (int dy = -1; dy <= 1; dy++)
        // 访问 (x+dx, y+dy)

DFS 本质：栈（递归）实现的深度优先搜索，沿着一个方向走到黑，再回溯。

2. slide_window — 单调队列求滑动窗口最值

问题：长度为 k 的滑动窗口在数组上移动，求每个窗口的最小值和最大值。

思想：单调队列

用双端队列（deque）存储下标，维护：
  · 最小值队列：递增（队首最小）
  · 最大值队列：递减（队首最大）

维护规则（以最小值队列为例）：

操作 条件 动作
队尾淘汰 a[新元素] <= a[队尾] 队尾弹出（新元素更小且更晚过期）
队首淘汰 队首下标 < i - k + 1 队首弹出（滑出窗口）
输出答案 i >= k - 1 队首即当前窗口最值 
void min_sliding() {
    deque<int> dq;
    for (int i = 0; i < n; i++) {
        while (!dq.empty() && a[dq.back()] >= a[i]) dq.pop_back();
        dq.push_back(i);
        if (dq.front() <= i - k) dq.pop_front();
        if (i >= k - 1) cout << a[dq.front()] << " ";
    }
}

3. tree_recover — 二叉树遍历恢复

问题：已知前序遍历和中序遍历，求后序遍历。

思想：分治递归

前序：[根 | 左子树 | 右子树]     根在第一个
中序：[左子树 | 根 | 右子树]     根分隔左右
                            ↓
      前序第一个字符 = 根
      在中序中找到根的位置 pos
      左子树长度 = pos - inL
                            ↓
      递归左子树 + 递归右子树 + 输出根（后序）
void post(int preL, int preR, int inL, int inR) {
    if (preL > preR) return;
    char root = pre[preL];
    int pos = inL; while (in[pos] != root) pos++;
    int leftLen = pos - inL;

    post(preL + 1, preL + leftLen, inL, pos - 1);     // 左
    post(preL + leftLen + 1, preR, pos + 1, inR);     // 右
    cout << root;                                      // 根
}

三种遍历总结：

遍历方式 顺序 根的位置
前序 根 → 左 → 右 第一个
中序 左 → 根 → 右 中间
后序 左 → 右 → 根 最后一个

Class 5 — BFS 状态搜索与 DFS 剪枝

1. divide_cola — BFS 可乐平分

问题：三个容器（S, N, M），初始 S 满，通过互相倒可乐，求平分到两个容器各一半的最小步数。

思想：BFS 状态空间搜索

状态 = (s, n, m) 三元组
初始状态 = (S, 0, 0)
目标状态 = 任意两个容器各为 half

每次操作 = 从容器 i 倒入容器 j
    倒出量 = min(源容器剩余, 目标容器剩余空间)
    新状态 = 更新后的 (s', n', m')

vis[s][n][m] 三维去重，避免重复访问
状态空间规模：101 × 101 × 101 ≈ 10^6，BFS 可接受

关键判断：S % 2 != 0 直接返回 NO（奇数不可平分）。

BFS 保证：第一次到达目标状态时步数最小。

2. maze_problem — BFS 迷宫最短路径 + 路径记录

问题：5×5 迷宫从 (1,1) 到 (5,5) 的最短路径并输出。

思想：BFS + 前驱节点记录

node trace[10][10];          // 记录每个格子的前驱坐标

trace[xx][yy].x = fx.front();
trace[xx][yy].y = fy.front();

路径输出（递归）：

void print(int x, int y) {
    if (x == -1 && y == -1) return;   // 起点
    print(trace[x][y].x, trace[x][y].y);
    printf("(%d,%d)\n", x, y);
}

BFS 第一次到达终点就是最短路径，这是由 BFS 逐层扩展的特性保证的。

3. number_transformer — BFS 数字变换

问题：数字 K 到 P 的最小操作次数，每次可选：×2、偶数÷2、+1。

思想：一维 BFS

状态是一维的（数字本身）
dist[x] 记录从起点到数字 x 的最短步数
三种操作生成三个新状态

注意：状态空间需要限制上界（这里 MAXN = 100000），操作也要防越界。

int nx = x * 2;
if (nx < MAXN && dist[nx] == -1) {
    dist[nx] = dist[x] + 1;
    q[tail++] = nx;
}

4. square — DFS + 剪枝拼正方形

问题：判断能否用所有火柴拼成一个正方形。

思想：DFS 逐边拼接 + 三大剪枝

target = 总长 / 4
降序排序 → DFS 逐根火柴放入当前边

三种剪枝：

剪枝 代码 效果
① 降序排序 sort(..., greater<>()) 长火柴先放，分支更少
② 第一根失败 if (sum == 0) return false 这根火柴放哪条边都不行
③ 等长跳过 while (i+1 < n && sticks[i+1] == sticks[i]) i++ 相同长度，结果相同 
普通 DFS：4^n 种可能
+ 三大剪枝：实际运行极快

Class 6 — DFS 回溯

prime_circle — 素数环

问题：1~n 排成一个环，相邻两数之和为素数，输出所有排列。

思想：DFS 回溯

固定 1 在第一位（旋转对称）
从 step=2 开始 DFS 每个位置
    ↓
尝试所有未使用的数字 i
检查 a[step-1] + i 是否为素数
    ↓
是 → 标记已用 → DFS 下一层 → 回溯撤销标记
void dfs(int step) {
    if (step > n && isprime(a[n] + 1)) {   // 检查首尾和
        // 输出解
        return;
    }
    for (int i = 2; i <= n; i++) {
        if (!vis[i] && isprime(a[step-1] + i)) {
            vis[i] = 1;
            a[step] = i;
            dfs(step + 1);
            vis[i] = 0;    // 回溯撤销
        }
    }
}

回溯模板：

DFS(当前状态) {
    if (到达边界) { 输出/记录; return; }
    for (所有可能的候选) {
        if (候选合法) {
            做出选择（标记）
            DFS(下一状态)
            撤销选择（取消标记）  ← 这是回溯的关键
        }
    }
}
