二、信息的表示与处理

进制转换

转换方向 方法
二进制 ↔︎ 十六进制 每 4 位二进制对应 1 位十六进制，从低位开始分组
十进制整数 → 二进制 除 2 取余，余数从下往上排列
十进制小数 → 二进制 乘 2 取整法，反复乘 2，取整数部分，直到小数部分为 0 或达到精度
二进制/十六进制 → 十进制 按权展开相加

陷阱： 十六进制转二进制时，高位多余的 0 可以直接省略。如 0x0A → 1010，不必写成 00001010。

位运算

运算符 含义 应用场景
& 按位与 掩码提取——保留某些位（如 x & 0xFF 取低 8 位）
\| 按位或 设置某些位为 1（如 x \| 0x01 将最低位置 1）
~ 按位取反 构造掩码（如 ~0xFF 得到除低 8 位外全 1）
^ 按位异或 翻转特定位、交换两个值（a ^= b; b ^= a; a ^= b）

移位运算

类型 对应指令 行为
逻辑左移 shl / << 左移，低位补 0，高位丢弃
逻辑右移 shr / 无符号数的 >> 右移，高位补 0
算术右移 sar / 有符号数的 >> 右移，高位补符号位（原最高位）

关系： x << k = x × 2^k（不溢出时）；无符号右移 x >> k = ⌊x / 2^k⌋（向零取整）；有符号算术右移需要对负数做偏置修正才能得到向零取整的结果。

字长

字长是 CPU 一次能并行处理的最大位数，也是通用寄存器的宽度。x86-64 的字长为 64 位，x86-32 的字长为 32 位。字长决定了虚拟地址空间的大小（64 位可寻址 2⁶⁴ 字节）。

字节序（Endianness）

  • 小端序（Little-Endian）： 低位字节存储在低地址。x86 架构使用小端序。
  • 大端序（Big-Endian）： 高位字节存储在低地址。网络协议（IP/TCP）通常使用大端序。

例：存储 0x12345678（4 字节）在地址 0x100：

小端序:  0x100: 78 | 0x101: 56 | 0x102: 34 | 0x103: 12
大端序:  0x100: 12 | 0x101: 34 | 0x102: 56 | 0x103: 78

整数编码

类型 编码规则
无符号数 编码直接等于二进制数值，范围 0 ~ 2^w - 1
有符号数（补码） 正数等于二进制数值；负数 = 按位取反再加 1

易错： 正数的补码就是其本身，不要看见有符号数就去取反加一！

补码计算示例（8 位为例）：

+5  = 00000101（直接就是二进制）
-5  = 11111011（5→00000101，取反→11111010，加1→11111011）

反过来验证：11111011 解读为有符号数
→ 最高位为 1 是负数，先减 1 → 11111010，取反 → 00000101 = 5 → 所以是 -5 ✓

整数类型转换与扩展

有符号与无符号转换

编码不变，重新解读——值可能改变。

int x = -1;               // 补码: 0xFFFFFFFF (32位)
unsigned u = x;           // u = 4294967295
// 位模式 0xFFFFFFFF 不变，但解读方式变了

低位宽扩展为高位宽

原类型 扩展方式 示例（8→16位）
有符号数 符号位扩展——新扩展的高位全补符号位 8位 11111011（-5）→ 16位 11111111 11111011（-5）
无符号数 零扩展——新扩展的高位全补 0 8位 11111011（251）→ 16位 00000000 11111011（251）

截断

高位部分直接丢弃，不区分数据类型。

int a = 0x12345678;
char c = a;          // 截断到 8 位: c = 0x78
// 按补码解析 0x78 → +120

整数运算

运算 规则
加法/减法 编码直接相加/相减，然后丢弃超出位宽的部分
减法转化 -x = ~x + 1——取反再加 1，这是补码的特性
乘法 两个 w 位的数相乘，结果理论上需要 2w 位，低 w 位是结果，高位可能溢出

例子（4 位无符号）：

  1101 (13)
+ 0111  (7)
----------
 10100 → 截断为 0100 (4)  → CF=1（溢出）

浮点数（IEEE 754）

32 位单精度（float）格式

S (1位) | exp (8位) | frac (23位)

64 位双精度（double）格式

S (1位) | exp (11位) | frac (52位)

三种编码类型

类型 特征 E 的计算 M 的计算 用途
规格化数 exp 不全为 0 且不全为 1 E = exp - bias，bias = 2^(k-1) - 1 M = 1 + frac（有隐含的 1） 常规数值
非规格化数 exp 全为 0 E = 1 - bias（固定值） M = frac（隐含位是 0） 接近 0 的数，等间距
无穷大 exp 全为 1，frac 全为 0 — — 运算结果溢出
NaN exp 全为 1，frac 不全为 0 — — 无效运算结果

注意： - 十进制转浮点数时，规格化数不要忘记隐含的前置 1 - 浮点数中有两个零：+0.0 和 -0.0（符号位不同），但比较时 +0.0 == -0.0 为 true - 浮点数在大数值时不等间距——指数越大，相邻可表示数的间隔越大

十进制转浮点数步骤（以 float 为例）

  1. 将十进制数转为二进制（整数部分除 2 取余，小数部分乘 2 取整）
  2. 规格化：写成 ±1.xxx × 2^E 的形式
  3. 计算指数编码：exp = E + bias
  4. 尾数：取 xxx...（去掉隐含的 1），低位补 0 到 23 位

浮点数与整数转换

转换 安全性 说明
int → double 安全 double 尾数 53 位 > int 的 31 位，所有 int 都能精确表示
int → float 可能有精度损失 float 尾数只有 24 位，|int| > 2^24 时会丢失精度
float/double → int 截断小数 向零取整；值超出 int 范围时为未定义行为（UB）

常用指令

数据传送指令

指令 操作大小 示例
movb 1 字节 movb $0xFF, %al
movw 2 字节 movw %ax, %bx
movl 4 字节 movl %eax, %ebx
movq 8 字节 movq %rax, %rbx

关键约束：mov 的两个操作数不能都是内存。 内存到内存的传送需要分两步：先读到寄存器，再写到目标内存。

mov 与 lea 的区别

movq (%rdi, %rsi, 4), %rax    ; 先计算地址，再读该地址的值到 rax
leaq (%rdi, %rsi, 4), %rax    ; 只计算地址表达式的值，不访存，结果即地址
  • mov：计算地址 → 访问内存 → 取数据
  • lea：只计算地址表达式（常用于做算术，不改变标志位）

lea 的妙用： 编译器常用 lea 做无副作用的算术运算。

; 计算 x * 5
leaq (%rdi, %rdi, 4), %rax    ; rax = rdi + rdi * 4 = x * 5

控制流

条件跳转指令（jXX）

cmp 或 test 设置标志位后，jXX 根据标志位判断是否跳转。

cmp S, D 的含义： 计算 D - S，设置标志位但不写回结果。

test S, D 的含义： 计算 D & S，设置标志位但不写回结果。

后缀 含义（有符号） 后缀 含义（无符号） 标志位条件
je / jne 相等/不等 相同 相同 ZF=1 / ZF=0
jg / jge 大于/大于等于 ja / jae Above SF=OF（且 ZF=0 时）
jl / jle 小于/小于等于 jb / jbe Below SF≠OF（或 ZF=1 时）

注意： jg 不是直接看 SF=0，而是 SF == OF。

条件数据传送（cmov）

cmovg %rax, %rbx    ; 如果大于（有符号），rbx = rax，否则 rbx 不变
对比项 jXX（条件分支） cmov（条件移动）
原理 猜测分支方向，猜错则清空流水线 两条路径都计算，条件满足才写回
预测准确率高（>95%） 快 较慢（两边都算）
预测准确率低（~50%） 慢（频繁猜错） 更快
适用场景 两侧有函数调用/副作用 简单数据选择

不适用 cmov 的情况： 可能引发异常的内存访问（如空指针解引用）、浮点数、目的操作数是内存。

流程分析技巧

读汇编找等价 C 代码时： 注意 if 判断条件是否反了。如果满足条件直接跳转到 ret 了，那 if 里面的条件应该是相反的。

// C 代码
if (x <= 0)
    return 0;

// 可能的汇编
    cmpl $0, %edi
    jg   .L1           ; 如果 x > 0，跳转到执行主体（不返回）
    movl $0, %eax
    ret
.L1:
    ...

这里 jg（大于则跳走）说明条件写的是”小于等于时返回 0”。

Switch 与跳转表

当 case 值密集连续时，编译器生成跳转表（.rodata 中的地址数组）：

switch (x) {
    case 0: ... case 1: ... case 2: ... default: ...
}
    cmpl   $2, %edi
    ja     .L_default
    jmp    *.L4(, %rdi, 8)    ; 间接跳转

    .section .rodata
.L4:
    .quad .L_case0
    .quad .L_case1
    .quad .L_case2

跳转表是 O(1) 的——不管多少个 case，执行时间固定。条件链是 O(n)。

过程（函数调用）

栈的操作

栈由高地址向低地址生长（RSP 递减）。

指令 效果
push %rax RSP -= 8，[RSP] = %rax
pop %rax %rax = [RSP]，RSP += 8
call func RSP -= 8，[RSP] = RIP，RIP = func（压入返回地址）
ret RIP = [RSP]，RSP += 8（弹出返回地址）

call 和 ret 的对应关系：

main:
    call func          ; 1. 把下一条指令地址（main+offset）压栈
    ...                 ; 2. 跳转到 func
                        ; 4. 返回到这里继续执行
func:
    ...
    ret                 ; 3. 从栈顶弹出返回地址，跳回

栈帧

每个函数的调用都要为其创建一个栈帧。函数的嵌套调用正好符合栈的结构——后调用的先返回。

func:
    push   %rbp        ; 保存调用者的 RBP
    mov    %rsp, %rbp  ; 建立当前函数的帧指针
    sub    $N, %rsp    ; 分配局部变量空间
    ...
    mov    %rbp, %rsp  ; 恢复 RSP
    pop    %rbp        ; 恢复调用者的 RBP
    ret

参数传递（x86-64 System V ABI）

参数位置 寄存器
第 1 个 %rdi
第 2 个 %rsi
第 3 个 %rdx
第 4 个 %rcx
第 5 个 %r8
第 6 个 %r9
第 7 个起 栈（从右向左压栈）

返回值： 通过 %rax 传递。

注意： 分析多参数函数时，将参数和寄存器按顺序对应起来即可，不需要死记硬背。

寄存器使用惯例

类型 寄存器 责任
Caller-saved %rax, %rcx, %rdx, %rsi, %rdi, %r8~%r11 调用者在 call 前自行保存需要的值
Callee-saved %rbx, %rbp, %r12~%r15 被调者负责在入口保存、返回前恢复

设计原则： 让最短生存期的数据由调用者管，最长生存期的由被调者管，使总体保存/恢复次数最少。

数组

  • 数组是一片连续的内存空间
  • 行优先（Row-Major）——每一行之间的数据是连续存储的
  • 多层数组相当于用指针把多个数组串起来

地址计算： 对于 int arr[R][C]，arr[i][j] 的地址 = arr_base + (i × C + j) × 4

结构体

  • 结构体也是一段连续的空间，区别在于包含不同类型成员
  • 通过相对于起始位置的偏移量来计算地址

字节对齐规则（重点，可能 10 分题）

  1. 每个类型的起始地址必须是其对齐要求的整数倍：
类型 大小 对齐要求
char 1 1（任意地址）
short 2 2（偶地址）
int / float 4 4
long / double / 指针 8 8
  1. 结构体的大小必须是其中最大成员对齐要求的整数倍（末尾可能有填充）

三条规则的递进关系：

规则 1（自然对齐）→ 决定规则 2 需要的倍数
规则 2（成员偏移对齐）→ 决定字段间的填充（padding）
规则 3（结构体大小对齐）→ 决定末尾填充，保证数组也能满足规则 2

示例：

struct S {
    char   c;     // offset 0, size 1
    // 填充 3 字节
    int    i;     // offset 4 (4 的倍数), size 4
    short  s;     // offset 8 (2 的倍数), size 2
    // 填充 2 字节（保证大小为 4 的倍数）
};
// sizeof(struct S) = 12

优化技巧： 按成员对齐要求从大到小排列，可以减少填充。

struct Bad { char c; int i; short s; };    // sizeof = 12
struct Good { int i; short s; char c; };   // sizeof = 8（少 4 字节填充）

代码注入攻击

基本原理

在缓冲区中写入多于缓冲区长度的数据，覆盖返回地址，操控控制流跳转到注入的恶意代码位置。

void echo() {
    char buf[4];        // 很小的缓冲区
    gets(buf);          // gets 不检查长度！
}
栈布局（溢出前）:
┌──────────────────┐
│ buf[0~3]          │ ← 缓冲区（4 字节）
├──────────────────┤
│ 填充区（20 字节）  │ ← gets 溢出时先写这里
├──────────────────┤
│ 返回地址           │ ← 被覆盖后，ret 跳到攻击者控制的位置
└──────────────────┘

缓冲区只是 buf[4] 那 4 个字节。溢出就是突破了这 4 个字节写到了外面。 填充区和返回地址不是缓冲区的一部分——它们是缓冲区溢出的受害者。

三种防御手段

防御手段 原理
栈随机化 每次运行栈起始地址随机变化（ASLR），攻击者难以定位注入代码的位置
不可执行位（NX bit） 标记栈内存不可执行。即使注入了代码，CPU 也无法执行它 → 崩溃而非被攻击
栈金丝雀（Stack Canary） 在栈帧中插入随机值（canary），函数返回前检查是否被修改。被破坏则调用 __stack_chk_fail 终止程序

四、处理器体系结构

CISC vs RISC

对比项 CISC（x86） RISC（ARM、RISC-V）
指令长度 变长（1~15 字节） 定长（通常 4 字节）
指令复杂度 单条指令可做复杂操作 单条指令做简单操作，复杂功能由多条组合
寄存器数量 少（x86-64: 16 个通用寄存器） 多（ARM64: 31 个、RISC-V: 32 个）
访存方式 多数指令可直接操作内存 只有 load / store 指令访存
解码器 复杂（大量晶体管，功耗高） 简单（功耗低）
功耗 高 低

为什么 RISC 功耗更低？ 1. 解码器简单（ARM 解码器晶体管约 x86 的 1/10） 2. 寄存器多 → 减少访存（访存能耗是 ALU 运算的几十倍） 3. Load-Store 架构 → 控制逻辑简单 4. 条件码更新可控 → 减少无效翻转

经典五级流水线

阶段 英文 操作
取指 Fetch 从内存读取指令，同时更新 PC（PC += 指令长度）
译码 Decode 译码指令，从寄存器堆读取操作数
执行 Execute ALU 执行算术/逻辑运算
访存 Memory 读/写数据内存（Load/Store 指令）
写回 Write Back 将结果写回寄存器堆

注意： 在顺序执行的 CPU 中，可能访问内存的指令阶段是取指（读指令）和访存（读写数据）。

时钟周期与吞吐量

时钟周期 = 流水线中最慢阶段的延迟
          （流水线划分时要尽量让各阶段均分）

单条指令执行时间 = 时钟周期 × 流水线阶段数

吞吐量 = 1 / 时钟周期（每秒钟完成的指令数）

当流水线段数 K 划分到极限时（将访问寄存器的操作也作为一个阶段），吞吐量接近理论最大值。

示例：

假设各阶段延迟: F=2ns, D=3ns, E=3ns, M=4ns, W=1ns
最慢阶段 = M（4ns）
时钟周期 = 4ns
吞吐量 = 1 / (4 × 10⁻⁹) = 250 MIPS
单条指令时间 = 4ns × 5 = 20ns

流水线对程序员透明

流水线处理器的执行不需要对程序进行修改——通过硬件自动实现的转发（Forwarding）、停顿（Stall）、分支预测等机制保证与顺序执行等价。这是流水线设计的基本目标。

控制冒险（分支预测）

分支指令导致流水线不知道该取哪条指令。

解决方案（从简单到复杂）：

  1. 预测不跳转： 默认继续取顺序下一条。猜错时冲刷错误指令，损失 2 周期
  2. 动态分支预测： 使用分支历史表（BHT），2-bit 饱和计数器，准确率 >95%
  3. 分支目标缓冲器（BTB）： 不仅预测是否跳转，还缓存目标地址

五、存储器层次结构

常用存储器对比

类型 实现 速度 容量 成本 主要应用
SRAM 6 个晶体管 最快（~1ns） 小（MB 级） 高 CPU 缓存（L1/L2/L3）
DRAM 1 晶体管 + 1 电容 较慢（~50ns） 大（GB 级） 低 主存
磁盘（HDD） 机械盘片 最慢（~ms） 极大（TB 级） 极低 大容量持久存储
SSD 闪存 介于 DRAM 和磁盘之间（μs 级） 大 中 高速持久存储

SSD 比磁盘快，但比 DRAM 慢。 SRAM 比 DRAM 快，但容量小、成本高。

磁盘

基本术语：

术语 含义
磁道（Track） 盘面上的同心圆
扇区（Sector） 磁道上划分的最小存储单元（通常 512B 或 4KB）
柱面（Cylinder） 所有盘面同一半径的磁道集合
寻道时间 磁头移动到目标磁道的时间
旋转延迟 盘片旋转到目标扇区的时间（平均 = 半圈旋转时间）
传输时间 读写数据的时间

容量计算：

磁盘容量 = 磁头数 × 每面磁道数 × 每道扇区数 × 每扇区字节数

注意： 结果是 GB（10⁹ 字节），不是 GiB（2³⁰ 字节）。

数据访问时间：

访问时间 = 寻道时间 + 旋转时间 + 传输时间

局部性原理

类型 定义 示例
时间局部性 一个数据被使用后，近期还可能被再次使用 循环内的变量反复访问
空间局部性 一个数据被使用后，附近地址的数据也可能被使用 数组的顺序遍历
  • 指令的顺序执行体现空间局部性
  • 循环体体现时间局部性（反复执行相同指令）

一定会考局部性原理的概念题。

高速缓存（Cache）

组织结构

类型 说明 特点
直接映射 每个内存块只能映射到 Cache 中唯一的行 简单，但冲突不命中高
组相联 每个内存块可映射到一组中的任意一行 折中方案，广泛使用
全相联 每个内存块可放在 Cache 中任意行 灵活，但硬件成本高

容量计算

Cache 总容量 = 行数 × (有效位 + 标记位 + 数据块大小)

Cache 友好代码的特征

  1. 利用空间局部性： 按行优先遍历多维数组（步长为 1），避免跨大步长访问
  2. 利用时间局部性： 反复使用同一数据（如累加器、循环不变量）
  3. 每次访问都尽量使用已加载到 Cache 中的数据

能够分析未命中率： 例如矩阵乘法中不同的循环嵌套顺序（ijk vs ikj vs kij）对 Cache 未命中率的影响。

平均访问时间

平均访问时间 = 命中时间 + 未命中率 × 未命中惩罚

六、链接

程序构建四阶段

阶段 输入 输出 主要工作
预处理 .c .i 处理 #include、#define、宏展开、删除注释
编译 .i .s 将 C 代码翻译为汇编代码
汇编 .s .o 将汇编代码翻译为机器码（目标文件）
链接 .o + 库 可执行文件 符号解析 + 重定位

符号解析

符号类型 定义 说明
全局符号 定义在当前模块，可被其他模块引用 非 static 的函数和全局变量
外部符号 引用在其他模块中定义的符号 使用其他文件中的函数/变量
局部符号 仅在当前模块内可见 带 static 的函数和全局变量

强符号与弱符号

类型 定义
强符号 已初始化的全局变量、函数定义
弱符号 未初始化的全局变量

同名全局符号链接规则（三步法）：

  1. 不允许有多个同名强符号→链接错误
  2. 如果有一个强符号和多个弱符号→选择强符号
  3. 如果有多个弱符号→随机选一个（危险！可能导致不易察觉的 bug）
// file1.c
int x = 5;        // 强符号
void func() {}    // 强符号

// file2.c
int x;            // 弱符号

链接结果：选择 x = 5（强符号覆盖弱符号）。

如果有以下情况：

// file1.c
int x = 5;

// file2.c
int x = 10;       // 另一个强符号 → 链接错误！

两个强符号冲突，链接器报错。
