矩阵乘法分块优化与缓存分析

前提参数

参数 值
元素类型 double，8 字节
Cache line 8 个 double = 64 字节
分块大小 B×B 个 double
总 Cache 容量（以 double 计） C，满足 3B² < C
矩阵大小 n×n，n >> B

内存布局约定

C/C++ 按 行主序（row-major） 存储矩阵。矩阵 A 的地址分布：

A[i][j] = A_base + (i * n + j) * 8

同一行内相邻元素地址连续，跨行跳 n 个元素。

Cache line 映射规则

硬件以 cache line 为单位从内存加载数据。对于起始地址为 addr 的 double：

所属 cache line 的起始地址 = addr - (addr % 64)    // 64 字节对齐

即：同一 cache line 覆盖一段 64 字节连续地址，存放 8 个连续的 double。

代码

朴素版（i-j-k）

for (i = 0; i < n; i++)
  for (j = 0; j < n; j++)
    for (k = 0; k < n; k++)
      c[i][j] += a[i][k] * b[k][j];

分块版

void mmm(double *a, double *b, double *c, int n) {
    int i, j, k;
    for (i = 0; i < n; i += B)
      for (j = 0; j < n; j += B)
        for (k = 0; k < n; k += B)
          for (i1 = i; i1 < i+B; i1++)
            for (j1 = j; j1 < j+B; j1++)
              for (k1 = k; k1 < k+B; k1++)
                c[i1*n+j1] += a[i1*n+k1] * b[k1*n+j1];
}

外层 (i, j, k) 遍历 B×B 块，内层 (i1, j1, k1) 计算两个 B×B 块的乘法：

C_block(i,j) = Σ A_block(i,k) × B_block(k,j)   (k_block)

第一部分：朴素版（i-j-k）逐次追踪

执行顺序

i=0:
  j=0:  k=0, 1, 2, ..., n-1  → 计算 c[0][0]
  j=1:  k=0, 1, 2, ..., n-1  → 计算 c[0][1]
  j=2:  k=0, 1, 2, ..., n-1  → 计算 c[0][2]
  ...
  j=n-1: ...
i=1: ...

Cache line 命中/失效逐次追踪

取 n=1024，追踪前几次迭代。令 A_base=0x0，B_base=0x800000（仅用来模拟 cache line 地址对齐）。

固定 i=0，j=0，k=0..n-1

k=0:  a[0][0]  → 地址 0x0        → ❌ 冷 miss → 加载 cache line [a[0][0..7]]  (0x0-0x3F)
      b[0][0]  → 地址 0x800000   → ❌ 冷 miss → 加载 cache line [b[0][0..7]]  (0x800000-0x80003F)
      c[0][0]  → 地址 0x1600000  → ❌ 冷 miss → 加载 cache line [c[0][0..7]]

k=1:  a[0][1]  → 地址 0x8        → ✅ 命中（和 a[0][0] 同 cache line）
      b[1][0]  → 地址 0x800400   → ❌ miss → 加载 [b[1][0..7]]  (每行 n=1024, 跨度 8192=0x2000)
      c[0][0]  → 已加载          → ✅ 命中

k=2:  a[0][2]  → ✅ 命中
      b[2][0]  → 地址 0x800800   → ❌ miss → 加载 [b[2][0..7]]
      c[0][0]  → ✅ 命中

k=3:  a[0][3]  → ✅ 命中
      b[3][0]  → 地址 0x800C00   → ❌ miss → 加载 [b[3][0..7]]
      c[0][0]  → ✅ 命中

...

k=7:  a[0][7]  → ✅ 命中 (同一条 cache line 的最后一个元素)
      b[7][0]  → ❌ miss → 加载 [b[7][0..7]]
      c[0][0]  → ✅ 命中

k=8:  a[0][8]  → 地址 0x40 → ❌ miss（进入了新的 cache line [a[0][8..15]]）
      b[8][0]  → ❌ miss → 加载 [b[8][0..7]]
      c[0][0]  → ✅ 命中

k=0..n-1 阶段小结：

数组 每 8 次 k 迭代 miss 次数 原因
A 1 次 行内顺序访问，每 8 个连续元素跨一条 cache line
B B 列 = n 次 每步访问 b[k][0]，k 每递增 1 就跳到下一行，每行一条新 cache line，全部 miss
C 1 次（第一次） 固定地址，之后放寄存器

这一阶段共 miss = 1 + n + 1 ≈ n+2 次。

固定 i=0，切换到 j=1，k=0..n-1

j=0→j=1 之间访问的数据量（k=0..n-1 完整一轮）：
  A: n 个元素，n/8 条 cache line
  B: n 行第 0 列，n 条 cache line
  C: 1 个地址
  ——————————
  合计 ≈ (9/8)n 条 cache line 的新数据被加载

如果缓存容量 < (9/8)n（n 很大时必然成立），
那么 k=0 时加载的 [b[0][0..7]] 已被驱逐。
j=1:
  k=0:  a[0][0]  → 地址 0x0  → ❓ 可能是命中或 miss（取决于缓存策略和 j=0 一轮后是否仍保留，保守按 miss）
        b[0][1]  → 地址 0x800008 → ❌ miss！
                 → 注意：b[0][1] 和 b[0][0] 在同一 cache line [b[0][0..7]] 里！
                 → 但这条 cache line 在 j=0 的 k=0..n-1 循环期间已被后续大量数据挤出了缓存
                 → 只能重新从内存加载
        c[0][1]  → ❌ 冷 miss（第一次访问 c[0][1]）

  k=1:  a[0][1]  → 可能 hit (如果 a 的 cache line 没被完全驱逐)
        b[1][1]  → ❌ miss → b[1][1] 和 b[1][0] 同 cache line，同样已被驱逐
        c[0][1]  → ✅ 命中

  k=2...n-1: 类似，b 的每一行都 miss

朴素版缓存行为总结

每来一个 j:
  b 的整列 n 个元素全部重新 miss
  原因：从 j 列到 j+1 列，间隔了 (9/8)n 条 cache line 的访问
       ＞ 缓存容量 → 第 j 列的数据已在期间被完全驱逐

对 j=0..n-1 的每个 j 重复 n 次 → B 被读了 n² 次，每次都 miss

总 cache miss 估算：

来源 每轮 j 的 miss 总 miss（三层循环）
A n/8 n² × n/8 = n³/8
B n n² × n = n³
C（第一次） 1 n²
合计 ≈ 9n³/8

第二部分：分块版逐次追踪

取 n=1024，B=64（块大小 64×64 = 4096 个 double = 32KB，三块合计 96KB）。

外层块迭代结构

i=0:          (块行 0)
  j=0:        (块列 0)
    k=0:      计算 C_block(0,0) += A_block(0,0) × B_block(0,0)
    k=64:     计算 C_block(0,0) += A_block(0,64) × B_block(64,0)
    k=128:    计算 C_block(0,0) += A_block(0,128) × B_block(128,0)
    ...
    k=n-64:   计算 C_block(0,0) += A_block(0,n-64) × B_block(n-64,0)
              此时 C_block(0,0) 已累加完所有 k_block → 最终结果
  j=64:       (块列 64)
    k=0:      计算 C_block(0,64) += A_block(0,0) × B_block(0,64)
    ...

下面追踪一个典型块迭代 (i=0, j=0, k=0) 的内部细节。

内三层循环的完整执行顺序

  i1=0:
    j1=0:   k1=0, 1, 2, ..., B-1
    j1=1:   k1=0, 1, 2, ..., B-1
    ...
    j1=B-1: k1=0, 1, 2, ..., B-1
  i1=1:
    j1=0:   k1=0, 1, 2, ..., B-1
    ...
  ...
  i1=B-1:
    ...

阶段一：i1=0（首行，B_block 从冷启动到完全填充）

子阶段 1.1：j1=0，k1=0..B-1（首列首次遍历——逐条 cold miss）

i1=0, j1=0:

  k1=0:  a[0][0]   → 地址 0x0         → ❌ cold miss → 加载 cache line L_A0 = [a[0][0..7]]
         b[0][0]   → 地址 0x800000    → ❌ cold miss → 加载 cache line L_B00 = [b[0][0..7]]
         c[0][0]   → 地址 0x1600000   → ❌ cold miss → 加载 cache line [c[0][0..7]]

  k1=1:  a[0][1]   → 0x8             → ✅ 同 L_A0 → hit
         b[1][0]   → 0x800400        → ❌ miss → 加载 L_B10 = [b[1][0..7]]
         c[0][0]                      → ✅ 命中（已在缓存）

  k1=2:  a[0][2]   → ✅ 同 L_A0 → hit
         b[2][0]   → 0x800800        → ❌ miss → 加载 L_B20 = [b[2][0..7]]
         c[0][0]                      → ✅ hit

  k1=3:  a[0][3]   → ✅ hit
         b[3][0]   → 0x800C00        → ❌ miss → 加载 L_B30 = [b[3][0..7]]
         c[0][0]                      → ✅ hit

  k1=4:  a[0][4]   → ✅ hit
         b[4][0]   → 0x801000        → ❌ miss → 加载 L_B40 = [b[4][0..7]]
         c[0][0]                      → ✅ hit

  k1=5:  a[0][5]   → ✅ hit
         b[5][0]   → 0x801400        → ❌ miss → 加载 L_B50 = [b[5][0..7]]
         c[0][0]                      → ✅ hit

  k1=6:  a[0][6]   → ✅ hit
         b[6][0]   → 0x801800        → ❌ miss → 加载 L_B60 = [b[6][0..7]]
         c[0][0]                      → ✅ hit

  k1=7:  a[0][7]   → ✅ hit（L_A0 最后一个元素）
         b[7][0]   → 0x801C00        → ❌ miss → 加载 L_B70 = [b[7][0..7]]
         c[0][0]                      → ✅ hit

  k1=8:  a[0][8]   → 0x40            → ❌ miss → 加载 L_A1 = [a[0][8..15]]
         b[8][0]   → 0x802000        → ❌ miss → 加载 L_B80 = [b[8][0..7]]
         c[0][0]                      → ✅ hit
  ...（以此类推）
  k1=B-1=63:
         a[0][63]  → ❌ miss → 加载 L_A7 = [a[0][56..63]]
         b[63][0]  → ❌ miss → 加载 L_B63_0 = [b[63][0..7]]
         c[0][0]                      → ✅ hit

j1=0 阶段统计：

数组 访问次数 miss 次数 命中率 说明
A B = 64 B/8 = 8 87.5% 每 8 个连续元素跨一条 cache line，首元素 miss，后 7 个 hit
B B = 64 B = 64 0% b[k][0]，k 递增 = 逐行扫描第 0 列，每步新 cache line，全部 cold miss
C B = 64 1（仅首次） 98.4% 固定 c[0][0]，第一次 cold miss 后一直 hit

子阶段 1.2：j1=1..7，k1=0..B-1（同列后 7 列——同一 cache line 横向命中）

i1=0, j1=1:

  k1=0:  a[0][0]   → ✅ 同 L_A0 → hit（和 j1=0 完全相同的地址！）
         b[0][1]   → 0x800008 → ✅ ✅ ✅ **HIT！！！**
                    → b[0][1] 和 b[0][0] 在**同一条** cache line L_B00 里！
                    → L_B00 在 j1=0, k1=0 时已加载，且在缓存中未被驱逐
         c[0][1]   → 0x1600008 → ❌ 可能 cold miss（第一次访问 c[0][1]）

  k1=1:  a[0][1]   → ✅ hit（同 L_A0）
         b[1][1]   → 0x800408 → ✅ **HIT！！！**
                    → b[1][1] 和 b[1][0] 在同一条 cache line L_B10 里
                    → L_B10 在 j1=0, k1=1 时已加载
         c[0][1]   → ✅ hit

  k1=2:  a[0][2]   → ✅ hit
         b[2][1]   → 0x800808 → ✅ **HIT！！！**（已加载的 L_B20）
         c[0][1]   → ✅ hit
  ...
  k1=63: a[0][63]  → ✅ hit
         b[63][1]  → 0x801C08 → ✅ **HIT！！！**（已加载的 L_B63_0）
         c[0][1]   → ✅ hit

同样，j1=2..7 也全部如此——对 B_block 的全部访问都是 hit，因为第 0 列加载进来的 cache line 同时覆盖了第 1~7 列。

j1=1..7 阶段统计（B 的访问堪称完美）：

数组 访问次数×列数 miss 次数 命中率
A 64×7 0（全同 L_A0..L_A7 范围内） 100%
B 64×7 0（全在同一条 cache line 内） 100%
C 64×7 7（每列第一个元素 cold） ~98.4%

子阶段 1.3：j1=8..15，k1=0..B-1（新的一组 cache line）

i1=0, j1=8:

  k1=0:  a[0][0]   → ✅ hit（还在缓存）
         b[0][8]   → 0x800040 → ❌ miss → 加载 L_B08 = [b[0][8..15]]
                    → b[0][0..7] 和 b[0][8..15] 不在同一条 cache line！
                    → 但 b[0][8..15] 是新 cache line，从未加载过
         c[0][8]   → ❌ cold miss

  k1=1:  a[0][1]   → ✅ hit
         b[1][8]   → 0x800440 → ❌ miss → 加载 L_B18 = [b[1][8..15]]
         c[0][8]   → ✅ hit
  ...（B 全部 cold miss）

  k1=63: b[63][8]  → ❌ miss → 加载 L_B63_8 = [b[63][8..15]]

j1=8 阶段：B 再次全部 miss（新的一组 8 列，新的 cache line）。

i1=0, j1=9..15:
  对 B 的所有访问全部 →
  ✅ **HIT！！！**（j1=8 时已加载了覆盖 b[k][8..15] 的 cache line）

以此类推：每 8 列为一组，组内第一列 B 全部 miss（B 条 cache line），后 7 列 B 全部 hit。

i1=0 全部结束后——B_block 的缓存状态

i1=0 共遍历了 j1=0..B-1 所有列。
B_block 共 B²/8 = 4096/8 = 512 条 cache line，全部被加载过至少一次。

此时缓存状态（假设缓存足够大，满足 3B² < C）：
  [已驻留]  A_block(0,0) :  B/8 = 8 条 cache line（行 0 的 64 个元素）
  [已驻留]  B_block(0,0) :  B²/8 = 512 条 cache line（全部！）
  [已驻留]  C_block(0,0) :  B/8 = 8 条 cache line（行 0 的 64 个元素，部分）

没被驱逐，因为 3B² < C，这三个块的总大小（3 × 8 × 512 = 3 × 32KB = 96KB）小于缓存容量。

阶段二：i1=1（第二行，B_block 完全命中阶段）

i1=1:

  j1=0, k1=0:  a[1][0]  → 0x2000 → ❌ 可能 miss（新行，新的 cache line L_A0' = [a[1][0..7]]）
               b[0][0]  → 0x800000 → ✅ ✅ ✅ **HIT！L_B00 还在缓存中！**
               c[1][0]  → ❌ 可能 cold miss

  j1=0, k1=1:  a[1][1]  → ✅ hit（同 L_A0'）
               b[1][0]  → 0x800400 → ✅ **HIT！L_B10 还在缓存中！**
               c[1][0]  → ✅ hit
  ...
  j1=0, k1=63: a[1][63] → ❌ miss（L_A7' = [a[1][56..63]]）
               b[63][0] → 0x801C00 → ✅ **HIT！L_B63_0 还在缓存中！**
               c[1][0]  → ✅ hit

  j1=1..7, k1=0..63:
               b[k][1..7] → 全部 ✅ **HIT！**（同一 cache line，L_Bk0 们都在）

  j1=8, k1=0..63:
               b[k][8]  → ✅ **HIT！**（L_Bk8 们也在，i1=0 时已加载）
  ...

i1=1..B-1：对 B_block 的全部 B×(B²) 次访问，0 次 miss。

分块版一次块迭代完整统计

内层循环阶段 A miss A hit B miss B hit C miss C hit
i1=0
j1=0（首列首遍历） B/8 B−B/8 B 0 1 B−1
j1=1~7（同组列） 0 7B 0 7B 7 7B−7
j1=8（新组首列） 0 B B 0 1 B−1
j1=9~15（同组列） 0 7B 0 7B 7 7B−7
…（每 8 列一组） 0 … B/组 7B/组 … …
i1=1~B-1（后 B-1 行）
j1=0（首列） (B-1)×B/8 (B-1)(B−B/8) 0 (B-1)B B-1 (B-1)(B-1)
j1=1~7 0 (B-1)×7B 0 (B-1)×7B 7(B-1) (B-1)(7B−7)
j1=8~… … … 0 (B-1)×(B²) … …
合计（一次块迭代） B²/8 B³−B²/8 B²/8 B³−B²/8 B²/8 B³−B²/8

对 B 的 miss 分析——最关键的优化

对 B_block 的总访问次数 = B×B×B = B³
总 miss 次数           = B²/8  （仅在 i1=0 时，每 8 列一组，每组首列触发 B 次 miss）
总 hit 次数            = B³ − B²/8

命中率 ≈ 1 − 1/(8B)
  B=64 时 ≈ 1 − 0.002 = 99.8%

对比朴素版中 B 的命中率 ≈ 0%（每步都 miss）。

第三部分：为什么结果不同——关键分析

朴素版：同样的 cache line 加载了，但被挤走了

朴素版 j=0 → j=1：

同一 cache line L_B00 = [b[0][0..7]]

第一次访问：j=0, k=0      → ❌ cold miss → 加载 L_B00
想再次使用：j=1, k=0      → ❌ miss → L_B00 已被驱逐

中间发生了什么？
  j=0 的 k=0..n-1 循环中，新加载的数据：
    A 行 0: n/8 条 cache line
    B 第 0 列: n 条 cache line（包括 L_B00，L_B10，...，L_Bn-1_0）
    C: 少量
  合计 ≈ (9/8)n 条 cache line

  L_B00 在此过程中经历了：
  ┌─────────────────────────────────────────────────┐
  │ k=0:  L_B00 加载                                    │
  │ k=1:  L_B10 加载 → L_B00 仍"新鲜"                    │
  │ k=2:  L_B20 加载 → L_B00 靠后                          │
  │ ...                                                  │
  │ k≈C: 新加载的 cache line 已填满缓存 → 某条旧数据被驱逐    │
  │        （取决于替换策略，FIFO/LRU/随机）                 │
  │ ...                                                  │
  │ k=n-1: L_B00 已被驱逐                                  │
  │ j=1, k=0: 需要 L_B00 → ❌ 不在 → 重新加载                │
  └─────────────────────────────────────────────────┘
  
  关键：L_B00 在被再次需要之前，中间经过了 ≈ n 条 cache line 的加载
        当 n > 缓存能容纳的 cache line 数时，L_B00 必然被逐出

  而且更糟糕的是，b[0][1] 明明和 b[0][0] 在**同一条 cache line** 里！
  但因为 L_B00 已经被驱逐了，连这个横向复用也完全浪费了。

分块版：同一条 cache line，赶上了

分块版 i1=0 → i1=1：

同一 cache line L_B00 = [b[0][0..7]]

第一次访问：i1=0, j1=0, k1=0  → ❌ cold miss → 加载 L_B00
再次使用：  i1=0, j1=1, k1=0  → ✅ hit
                （中间隔了 B=64 次 k1 访问，仅加载了 A 的 8 条 cache line）
再次使用：  i1=0, j1=2, k1=0  → ✅ hit
再次使用：  i1=0, j1=7, k1=0  → ✅ hit
                （B=64 行，每 8 列一组，同组内全 hit）
再次使用：  i1=1, j1=0, k1=0  → ✅ hit
                （中间隔了一整行 i1=0 的 B×B 次访问，但 B_block 总共只占
                  B²/8 条 cache line，且 3B² < C，不会被逐出）
再次使用：  i1=1, j1=1, k1=0  → ✅ hit
...
           i1=B-1, j1=B-1, k1=B-1 → ✅ hit
                （从第一次加载到最后一次使用，L_B00 一直驻留在缓存中）

  L_B00 在此过程中：
  ┌──────────────────────────────────────────────────┐
  │ i1=0, j1=0, k1=0: L_B00 加载                          │
  │ i1=0, j1=0, k1=1..63: A 和 B 的更多 cache line 加载      │
  │ i1=0, j1=1..7, k1=0..63: 全部 hit，无新加载                 │
  │ i1=0, j1=8, k1=0: B 的下一组 cache line 加载               │
  │      （替换策略可能逐出最旧的缓存行，但 3B² < C，             │
  │       即使逐出也是逐出同块内的其他行，L_B00 仍可能被保留，      │
  │       或被逐出后会在同一 i1=0 轮被重新加载一次）              │
  │ ...                                                     │
  │ i1=1..B-1: L_B00 全部 hit                                │
  └──────────────────────────────────────────────────┘

  关键：L_B00 第一次加载后到最后一次被使用，
        中间只经过了约 B²/8 条 cache line 的加载
        因为 3B² < C，这些数据**撑不满缓存**
        → L_B00 得以全程驻留

本质差异对照

朴素版 分块版
B 的 cache line 加载时机 j=0, k=0..n-1 逐行 cold miss i1=0, j1=0, k1=0..B-1 逐行 cold miss
加载的 cache line 数量 每条 B 的 cache line 加载 n 次（每列一次） 每条 B 的 cache line 加载 1 次（i1=0 时加载，后 B-1 行复用）
同 cache line 横向复用 浪费：b[0][j] 和 b[0][j+1] 同 cache line，但整条 line 已被驱逐 充分利用：b[0][j] 加载的 cache line 覆盖 b[0][j+1..j+7]，后 7 列全命中
纵向复用 无：b[k][j] 加载后下一行就跳到新 cache line，无横向无关的复用 充分：i1=0 加载全部 B_block，i1=1..B-1 对 B_block 全部命中
从加载到末次使用的间隔 n 次访问（被驱逐） B² 次访问（不超缓存容量）
B 的总 miss n³ n³/(8B)（降低了 8B 倍）

第四部分：完整数据流总结

一次块迭代 (i, j, k) 内的数据流

数据从哪里来（内存 → 缓存）：
  
  A_block(i,k)：i1 第一次访问各行时，该行的 cache line 按需加载
                 每行 ≤B/8 条 cache line
                 行内后续访问 hit

  B_block(k,j)：i1=0 时，每组 8 列的首列触发 cold miss，
                加载该列的 B 条 cache line
                同组后 7 列全部 hit
                i1=1..B-1 时全部 hit

  C_block(i,j)：随 i1,j1 首次访问按需加载
                同一位置在 k1 循环中反复累加（寄存器）

数据流经路径：
  内存 → L_cache（加载整条 cache line）→ CPU 寄存器（计算）→ L_cache（写回）
  一个 cache line 从内存加载后，在缓存中被访问多次（读/写）后才可能被逐出

缓存不命中类型

miss 类型 出现场景 分块版 vs 朴素版
Cold miss（强制失效） 第一次访问某条 cache line 相同：分块版只是将 cold miss 集中到 i1=0 阶段
Capacity miss（容量失效） 工作集 > 缓存容量，之前加载的 cache line 被后续数据挤走 分块版消除：3B² < C 保证工作集不超容量
Conflict miss（冲突失效） 因缓存关联度限制，两个地址映射到同一缓存组相互驱逐 两者都可能发生，分块版可通过调整 B 缓解

总 cache miss 估算对比

版本 对 A 的总 miss 对 B 的总 miss 对 C 的总 miss 总 miss
朴素 n³/8 n³ n² 9n³/8
分块 n³/(8B) n³/(8B) n²/B n³/(4B)（首项 n³ 项）
提升倍率 8B 倍 8B 倍 B 倍 ≈ O(B) 倍 
以 B=64 时：分块版的 B miss 从 n³ 降到 n³/(512)，降低 512 倍！

第五部分：何时需要重新加载

分块版中 B_block 的 cache line 只需要加载一次，前提是 3B² < C。如果 B 选得太大，导致 3B² > C：

B×B 块过大 → 3B² > C → A/B/C 三个块无法同时驻留
→ 在 i1=0 加载的 B_block 会在 i1 还没遍历完时就被挤出去一部分
→ i1=1 时发现部分 cache line 已被驱逐 → 重新 miss
→ 退化为类似朴素版的性能

所以 B 的选择必须满足 3B² < C，这是分块有效的硬性条件。
